# UI 设计方案 v2.0 — AI 求职一站式小程序

本目录为「AI 求职一站式」微信小程序的**全新 UI 设计方案 v2.0**（UiDesigner 出品），在原有 `ui-design-spec-miniapp-ai-2026-07-07.md`（v1.0 绿紫工具化扁平规范）基础上重构，聚焦**美观度 + 可用性 + 可落地性**。

## 文件清单

| 文件 | 说明 |
|------|------|
| `ui-design-v2-miniapp-ai-2026-07-07.html` | **主交付物**：自包含 HTML。包含完整设计系统（色彩 / 字体 / 间距 / 圆角 / 阴影 / 组件库）、9 屏手机高保真 Mockup、逐屏交互细节标注、动效与手势规范、无障碍基线。浏览器直接打开即可滚动浏览。 |

## 相较 v1.0 的关键升级
- 双主色 → **主次分明有语义**：翠绿 `#16A36E` 为主（证件照/主操作），紫罗兰 `#6C5CE7` 为辅（AI/简历）。
- 纯色扁平 → **渐变质感 + 留白 + 柔和阴影**，让"全免费"也有专业信任感。
- 令牌偏少 → **完整令牌系统**（6 级色彩 / 6 级字号 / 4 级圆角 / 3 级阴影 / 4px 网格），可 1:1 映射 WXSS。
- 文字描述组件 → **可视化组件库 + 9 屏 Mockup + 逐屏交互细节**。
- 状态散列 → **统一四态规范**（加载 / 错误 / 空态 / 破坏操作确认）。
- 类名衔接：HTML 用 `--brand` 短名，WXSS 统一加 `--c-` 前缀（`--c-brand`）适配小程序命名空间；组件类名两边完全一致，开发可 1:1 对照实现。

## 落地建议
以 `pages/photo/edit` 证件照编辑流为最高优先级核心页，先建 `styles/` 引入两个 wxss，再逐页套用令牌。分享卡片、合规 6 项评分环、AI 润色对比为差异化亮点，建议优先打磨。
