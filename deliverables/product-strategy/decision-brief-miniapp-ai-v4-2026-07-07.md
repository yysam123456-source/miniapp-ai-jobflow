## 证件照处理工具链 · 完整设计

> 证件照功能从原 Spec 的 F2（职业照生成）/F3（合规检测）/F4（抠图换底）扩展为四大类共 9 项子能力：**AI 职业照生成** · **图像编辑**（抠图+去背景+去水印+修图）· **合规检测** · **规格导出**。全部基于免费前提落地。

### 1. AI 职业照生成（F2a/F2b）

基于免费生图额度（10 万张 Hy Image 3.0），提供"上传生活照→生成专业职业照"能力。

| 步骤 | 功能 | 实现 |
|------|------|------|
| F2a | 人像 Prompt 生成 | Hy3 分析用户照片特征（性别/发型/服装风格/场景），生成优化后的 Hy Image 3.0 prompt |
| F2b | AI 职业照生成 | Hy Image 3.0 文生图，输出专业职业照（正装/商务背景/柔和打光） |

**关键约束**：10 万张生图额度 → 按 "生成预览 + 用户确认后出高清" 两步走，非确认不消耗额度。预估每用户消耗 1-3 张（预览1张+最终1-2张）。

### 2. 图像编辑工具链（F3a–F3d）

全部跑在**云函数内自托管免费开源模型**，零 API 调用费，仅消耗云函数免费 CPU 运行时。

| 子功能 | 能力 | 免费实现方案 | 技术细节 |
|--------|------|-------------|----------|
| **F3a 抠图** | 人像主体提取 → 输出透明 PNG | `rembg`（Python onnxruntime）或 `@imgly/background-removal`（Node.js，评估开源许可）自托管云函数层 | 输入 ≤ 2048×2048，输出透明通道 PNG；发丝级精度取决于模型版本（u2net/isnet）；冷启动首张 < 5s，后续 < 2s |
| **F3b 去背景** | 透明图 → 替换纯色/自定义背景 | 复用 `craftisle-app` 的 `BG_COLORS`（白/蓝/红/浅蓝/浅灰）和 `applyBackground` 纯几何换色 | 基于 F3a 结果；Sharp 或 Canvas compositing；支持自定义上传背景图 |
| **F3c 去水印** | 手涂 mask → AI 智能填充消除水印/文字/logo | 自托管开源 inpainting（LaMa / lama-cleaner 轻量版），onnxruntime-node 运行 | 用户在小程序端手指涂抹水印区域 → mask 上传云函数 → inpainting 模型填充 → 返回修复图；证件照场景图片小（~1MP），模型推理 < 10s |
| **F3d 修图** | 亮度/对比度/饱和度/色温/锐化/磨皮/去瑕疵 | **基础修图**：Sharp npm 零成本传统图像处理（`modulate` / `linear` / `sharpen`）；**磨皮**：onnx 轻量人脸美化模型自托管（已确认） | 基础修图即时（纯像素操作）；磨皮模型 < 50MB，推理 < 3s（证件照尺寸）；皮肤检测+高斯模糊遮罩亦可纯算法实现磨皮 |

**UI 交互**：编辑工具栏横排（抠图 → 去背景 → 去水印 → 修图 → 完成），每步实时预览。修图项支持滑块调节（亮度/对比度/饱和度/锐化）。去水印支持手指涂抹标记区域。

### 3. 合规检测（F3e）

| 检测项 | 说明 | 免费实现 |
|--------|------|----------|
| 人脸检测 | 是否有完整人脸、是否正面 | 开源 face-api.js / MediaPipe（onnxruntime） |
| 姿态检测 | 是否歪头/侧脸 | 同上，计算人脸关键点偏角 |
| 光线检测 | 是否过暗/过曝/阴阳脸 | 直方图分析（纯 JS/Sharp） |
| 遮挡检测 | 眼镜反光/头发遮脸/口罩 | 分割模型 + 规则 |
| 背景检测 | 是否复杂背景/有人 | 抠图结果 → 计算背景复杂度 |
| 尺寸检测 | 像素是否达标 | 直接读宽高 |

每项返回 `{pass:boolean, score:number, suggestion:string}`。不通过 → 红色标记 + 修正指引（如"请摘掉眼镜""面部光线不足，请面向光源"）。

### 4. 规格导出（F4a/F4b）

| 子功能 | 能力 | 复用来源 |
|--------|------|----------|
| **F4a 规格裁切** | 1寸/2寸/护照/美签/英签/申根 6 规格一键裁切 | `craftisle-app`: `ID_PHOTO_SIZES` + `cropToSize` |
| **F4b 回执排版** | 按城市拼版（如深圳出入境回执）+ 回执文本叠印 | 新建；Sharp compose 多图排版 + 文字绘制 |

### 5. 工具链编排（用户操作流）

```
📷 选择照片
    ├─ 📐 AI 职业照生成（可选，走 F2a→F2b）

          ├─ ✂️ 抠图（自动提取人像）
          ├─ 🎨 去背景（选色 / 自定义背景）
          ├─ 🧹 去水印（手涂标记 → AI 消除）
          ├─ ✨ 修图（亮度/对比度/饱和度/锐化/磨皮）
          ├─ ✅ 合规检测（6 项逐项评分）
          └─ 📤 导出（选规格/回执 → 下载/分享）
```

## 四、免费前提下的真实约束（设计必须面对，而非"付费选项"）

免费≠无约束。真正要工程上处理的是**额度约束**，不是成本：

1. **6 个月有效期**：资源包申请成功起 6 个月有效 → MVP 必须在此期间跑通验证，并尽早启动流量主变现以建立"自给"路径。
2. **额度上限（10 亿 token / 10 万张生图）**：需**额度监控**（按用户/按日计量，防超额转付费）；对高耗接口（importParse、optimize）做缓存与限流。
3. **云函数/存储/数据库基础配额**：个人版资源点套餐有免费上限 → 中文字体体积、PDF 渲染冷启动需控制；素材定期清理。
4. **内容安全**：微信侧内容安全检测（免费基础能力）必接，拦截率目标 <2%。

> 这些约束 100% 可在免费额度内解决，不构成"要不要付费"的拍板项。
