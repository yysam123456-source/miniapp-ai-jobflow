## 📌 TL;DR（执行摘要）

- **产品**：微信小程序「AI 求职一站式」——把 AI 职业照生成、证件照编辑（抠图/去背景/去水印/修图/合规检测/规格导出）、简历制作与 AI 润色、PDF/Word 导出串成一条向导式流水线。
- **商业模式**：全免费——基于微信云开发 CloudBase + 微信 AI 小程序成长计划（10 亿 token + 10 万张生图，混元 Hy3 + Hy Image 3.0，6 个月免费），靠微信流量主广告反哺长期；零服务器/域名/ICP 成本。
- **MVP 范围**：简历工作流（F1/F6/F6b/F6c/F7/F7b）+ 证件照工具链（F2a/F2b/F3a–F3e/F4a/F4b）+ 双入口（F5）+ 分享（F8）+ 登录/历史（F9）+ 流量主变现。
- **代码复用**：简历核心 80% 逻辑直接复用 craftisle-resume（schema/resume/pdf/fonts/docx/import），AI 层换 `wx.cloud.extend.AI` 混元，prompts/sanitize 零改。
- **证件照工具链全新建**：9 项子功能全部基于云函数自托管开源模型落地，零外部付费。

## 1. 产品目标（3 个清晰、正交的目标）

1. **一站式闭环**：用户进入小程序 → 拍/选照片 → 编辑证件照 → 导入简历 → AI 润色 → 导出 PDF → 微信分享给 HR。全链路无需跳出微信。
2. **全免费壁垒**：所有 AI/生图/图像编辑能力全部落在微信云开发免费额度内，竞品无法匹敌的零成本架构。
3. **复用撬动效率**：复用 craftisle-resume 已跑通的 PDF/DOCX/import/ai prompts/sanitize 生产线，证件照链路用开源模型补齐，MVP 工期控制在可执行范围。

## 3. 用户研究洞察

> 基于早期研究产出（瑞思，2026-07-07 交付）及竞品分析 PDF。

- **核心痛点**：简历制作与证件照是两个独立流程，用户需分别使用 2-3 个小程序/App 才能完成"简历+照片"求职材料包。
- **价格敏感**：竞品证件照免费版导出带水印且清晰度打折，简历模板优质款需付费。用户期望"全免费 + 高品质"。
- **移动优先**：70%+ 用户在手机上完成简历制作，证件照直接用手机拍摄。
- **分享驱动**：微信原生的分享能力是获客核心渠道（"帮我看看我的简历" → 裂变）。
- **流失点**：证件照不合规（被出入境窗口退回）导致用户投诉；简历 AI 润色"虚构内容"导致用户不信任。

## 5. 数据依据

- **市场规模**：2026 年国内 AI 小程序用户突破 3 亿（年增 25%）；小程序广告市场预计 850 亿元。
- **求职刚需**：2026 年高校毕业生预计 1270 万；社招换工作群体约 8000 万/年。
- **目标指标**：首月 DAU ≥ 3 万，WCLU（周闭环用户，完成一次证件照+简历全流程）≥ 8 万。
- **免费额度约束**：
  - 10 亿 token（文本 AI）= 按每用户完整简历流程消耗 ~50K token 估算 → 可支撑 **200 万** 用户
  - 10 万张生图 = 按每用户 2 张（预览+最终）估算 → 可支撑 **5 万** 用户
  - **瓶颈在生图额度**：需监控 + 限流，流量主收入可购买额外生图额度

## 7. 关键流程

### 7.1 求职简历工作流（F1）

```
[开始] → 选择照片 → [证件照处理子流程] → 导入简历（多种格式）
    → AI 解析填充 → 选择模板 → 预览/元素编辑
    → AI 润色（可选）→ 导出 PDF/Word → [分享/下载/保存]
```

### 7.2 证件照处理子流程

```
[选择/拍照] → [AI职业照生成(可选)] → 抠图 → 去背景(选色)
    → 去水印(可选) → 修图 → 合规检测
    → [不合格: 修正指引] / [合格: 规格裁切 + 回执排版] → 导出
```

### 7.3 数据流架构

```
┌────────────────── 微信小程序前端 ──────────────────┐
│  wx.login → openid                                  │
│  wx.cloud.uploadFile → 云存储                        │

│  onShareAppMessage 分享                              │
└──────────────────────┬──────────────────────────────┘
                       │ wx.cloud.callFunction
┌──────────────────────▼── 微信云开发 CloudBase ───────┐
│  ┌──────────────── 云函数 ─────────────────────┐     │
│  │  resume/    │  ai/       │  photo/          │     │
```

## 10. 技术风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 混元 API 接入方式未确认 | R3 阻塞 | T0 当天验证，一日可决 |
