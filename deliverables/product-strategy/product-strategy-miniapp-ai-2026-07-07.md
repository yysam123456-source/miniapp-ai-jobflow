- **核心目标**：在微信小程序内打通「证件照生成 ↔ 简历制作」端到端闭环，成为首个串联两节点的产品，借免费混元 + 现有代码复用实现零边际成本交付与微信原生裂变。

## 一、产品规划（主理人综合）

### 1.1 战略定位
微信小程序生态内首个将「证件照生成」与「简历制作」串联的一站式求职工作流产品。不重复造轮子——深度复用现有 `craftisle-resume`（简历内核）、`craftisle-app`（图片处理）代码资产，以混元 Hy3 + Hy Image 3.0 免费额度实现近乎零边际成本。

### 1.2 三层价值主张
1. **用户层**：不离开微信、信息只录一次、免费拿到过审证件照 + 可投递简历。
2. **商业层**：免费混元降本 + 流量主广告（≤1 次插屏，保体验）+ 微信原生分享裂变。
3. **工程层**：复用已验证业务逻辑，降自研 bug 与冷启动风险，压缩上线周期。

### 1.3 范围与边界（与 web 端 Craftisle 的关系）
- 代码可共用（同内核），但**账号/数据完全隔离**（Spec 锁定 R5），小程序侧独立埋点库，不并入核心指标。
- 首版不引入付费会员、不做多语言、不做图文创作流 / 自由画布 / UGC 模板市场。

### 2.3 市场数据 & 趋势（数析补充）
国内 AI 小程序用户 **3 亿**（年增 25%）、小程序月活 **10 亿**、广告市场 **850 亿**、微信成长计划半年 **4.5 万**小程序（70%+ 个人开发者）、**2026 高校毕业生 1270 万**。趋势：微信是最大增量场但简历/证件照竞品偏 web 未吃裂变；毕业季双刚需叠加，串联可借社交分享低成本获客。

### 2.4 SWOT
- **S**：免费混元降本；串联/双入口/微信分享差异化；现有代码资产快速兑现。
- **W**：首版模板仅 5 套；证件照引擎"非免费额度"需成本评估；云函数运行时兼容风险。
- **O**：3 亿用户增量场、1270 万毕业生刚性需求、串联位空缺。
- **T**：合合照 600+ 规格 / 100分 3000+ 模板体量压制；第三方引擎成本与合规风险。

### 2.5 差异化机会（代码复用落地证据）
差异化不止"设计主张"，而是**有现成资产可快速兑现**（codegraph+semble 已验证）：
- 串联/元素级编辑/导入映射/双入口/微信分享 → `craftisle-resume` 支撑（`createResumePdfFile` F7、`buildDocx` F7b、`sanitizeAndParseResumeJson` F6c、importers F6b、`resumeDataSchema`+`applyResumePatches` F6）。
- F4 后处理 → `craftisle-app` 支撑（`passportPhoto()` Sharp 换底/裁剪/导出含 China/US/UK/EU 50+ 规格、`id-photo`/`remove-bg` 抠图、`colorAdjust` 等 17+ 原语）。**修正旧结论"证件照零复用"。**
- 关键杠杆：`packages/ai` 已支持 `openai-compatible`，若混元 Hy3 暴露兼容协议（R3），则 AI 链路仅改 provider 配置。

## 三、需求设计 / PRD v1.0（析客 / Specky）

### 3.1 产品目标（3 个正交目标）
- **G1 占据市场空缺·一站式串联**：微信内打通证件照↔简历闭环（F1/F5/F8）。
- **G2 零成本高质量交付**：混元免费额度 + 现有代码复用，达标级润色/导出/合规（F6/F6b/F6c/F7/F4）。
- **G3 微信原生裂变增长**：原生分享（F8）+ 双入口续接（F5）沉淀资产（F9），北极星 WCLU 驱动转化。

### 3.2 用户故事（对应 Spec §4 A/B/C）
- **US1 场景 A·纯证件照**：办事考证人群免费拿过审照，选规格→生成（F2）→换底导出（F4）→合规（F3）→分享，不离开微信。
- **US2 场景 B·完整求职流**：应届/社招从一张照到可投递简历，选模板（F6）→导入填充（F6b）→润色（F6c）→生成照（F2/F4）→导出（F7）→分享（F8）。
- **US3 场景 C·双入口续接**：碎片化学生今天拍照、改天续做简历，进度不丢（F5）。
- **US4 社招提效**：1–3 年职场人 STAR 改写 + JD 匹配（F6c）。

### 3.3 用户研究洞察（瑞思）
- 画像：（a1）1270 万毕业生；（a2）25–35 岁跳槽；（b）教资/国考/签证备考。
- JTBD：JB1 一站式求职、JB2 零成本合规照、JB3 信息不重录、JB4 微信分享、JB5 双入口续接、JB6 资产沉淀。
- 优先级（Kano/MoSCoW）：F1–F7/F5/F6b/F6c/F9 = Must；F7b/F6d = Should；F6e = Could。
- 5 条差异化洞察：免费混元零成本、微信原生裂变、串联低摩擦、双入口降决策压力、信息一次录入全链路复用。

### 3.4 数据依据（数析）
北极星 **WCLU**（周完成闭环用户数）；漏斗基准 8%（区间 3.5–16.8%）；90 天目标 DAU≥3 万 / 月 WCLU≥8 万 / 7 日留存≥25% / 单用户 AI 成本≤0.05 元。
