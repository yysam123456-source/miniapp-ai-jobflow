## 3. 腾讯云账号 + CloudBase 环境 + 免费 1 亿 token（核心，必须看）

> 这一 Part 就是你之前问「免费 1 亿 token 在哪」和「为什么部署失败」的答案。

### 3.1 先搞清楚：你之前给的密钥为什么不行

你之前发给我的（密钥已脱敏，真实值只在本地 `config.local.js`，不进库）：
- 微信小程序 id：`wxf5fcd828dc79d7ba`
- 云环境：`cloudbase-d9g4sl8ab99ed3fd0`
- 混元 `SecretId=AKIDxxxx（本地已填）` / `SecretKey=********（本地已填）` / `AppId=1342223807`

我实测过：
1. 用这组密钥 `tcb login` **登录成功**（密钥有效）
2. 但 `tcb env list` 显示**该账号下零个 CloudBase 环境**
3. 对 `cloudbase-d9g4sl8ab99ed3fd0` 操作报 `[DescribeEnvInfo] An error has occurred`

**结论**：这组密钥是**混元专用子账号密钥**（大概率只挂了 `hunyuan:*` 权限），它认证的腾讯云账号 **≠ 拥有 `cloudbase-d9g4sl8ab99ed3fd0` 的账号**。
→ 所以：部署看不到环境、免费 token 也领不到这个环境里。这就是「账号对不上」。

### 3.2 免费 1 亿 token 从哪来（不是混元控制台自动给的）

那是「**微信 AI 小程序成长计划**」的补贴，包含：**6 个月免费云开发环境 + 1 亿 Token 混元 2.0 额度 + 1 万张文生图额度**。需要**报名 + 领取到指定账号和环境**才有。

**领取步骤（关键：选对账号 A 和环境）**：
1. 登录小程序后台 [mp.weixin.qq.com](https://mp.weixin.qq.com/)

3. 进到「**开发上线**」阶段 → 点领取「**AI Coding 云开发资源 + 混元免费 Token**」
4. 跳转到腾讯云登录页 → 弹窗里**选择领取的账号和环境**
   - ⚠️⚠️⚠️ **这里选的账号，必须就是创建 `cloudbase-d9g4sl8ab99ed3fd0` 的那个腾讯云账号 A；环境选 `cloudbase-d9g4sl8ab99ed3fd0`**

### 3.4 怎么确认「账号 A」到底是哪一个（决定你能不能部署）

`cloudbase-d9g4sl8ab99ed3fd0` 这个环境，是你当初在**微信开发者工具**里点「云开发」开通时创建的。当时登录的那个腾讯云账号就是 A。找回方法（任选）：

- **方法 1（最准）**：微信开发者工具 → 云开发 → 环境设置 → 右上角通常显示环境绑定的腾讯云 UIN/账号；或左侧「费用」/「账号中心」能看到当前登录的腾讯云账号。
- **方法 2**：拿你手头**所有可能的腾讯云账号**分别登录 [console.cloud.tencent.com](https://console.cloud.tencent.com/) → 云开发 → 看环境列表里有没有 `cloudbase-d9g4sl8ab99ed3fd0`。**有它的那个就是账号 A**。
- **方法 3**：如果实在分不清，最干脆——**在账号 A 里重新建一个云开发环境**（或就用现有环境），然后统一用这个账号的密钥。但注意：免费 token 必须领到「部署用的那个环境」所属账号。

> 💡 实操建议：直接拿**你登录微信开发者工具开通云开发时用过的那个腾讯云账号**去报名领取 + 拿密钥，100% 对得上。

## 5. 一键部署（bash deploy.sh）

脚本会：① 同步 `_shared` 到 15 个云函数 → ② 部署全部云函数（云端装依赖，避免 mac/linux 二进制冲突）→ ③ 建 5 个集合 `user/resume/asset/quota/workflow`。

### 方式甲：环境 API 密钥（推荐，你给我密钥即可）
```bash
CLOUDBASE_API_KEY=<环境API密钥> bash deploy.sh cloudbase-d9g4sl8ab99ed3fd0
```

### 方式乙：账号级密钥（你本地跑，或给我账号 A 的密钥）
```bash
# 先登录（用账号 A 的密钥）
tcb login --apiKeyId <账号A_SecretId> --apiKey <账号A_SecretKey>
# 再部署
bash deploy.sh cloudbase-d9g4sl8ab99ed3fd0
```

> 脚本已加 `--yes`，全程无人值守。CLI（`@cloudbase/cli`）需先装：`npm i -g @cloudbase/cli`（或本地 `npx @cloudbase/cli`）。
