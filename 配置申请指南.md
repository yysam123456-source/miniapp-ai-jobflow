## Part A — 微信平台资质（审核必过）

> 这些在 [mp.weixin.qq.com](https://mp.weixin.qq.com) 的「设置」→「基本设置」中完成，不填无法提交审核。

### A1. 注册小程序账号 → 获取 AppID

| 步骤 | 操作 |
|------|------|
| 1 | 浏览器打开 [mp.weixin.qq.com](https://mp.weixin.qq.com/)，右上角「立即注册」→ 选择「小程序」 |
| 2 | 填写邮箱（未注册过公众号/小程序的）、密码 |
| 3 | 邮箱收到激活链接 → 点击激活 |
| 4 | **主体类型选择**：个人选「个人」（只需身份证 + 手机号）；企业选「企业」（需营业执照 + 对公账户打款验证） |
| 5 | 个人主体：填姓名、身份证号、手机号，用绑了银行卡的微信号扫码验证管理员身份 |
| 6 | 完成注册 → 左侧「开发」→「开发管理」→「开发设置」→ 复制 **AppID** |

- **耗时**：10 分钟
- **结果**：`wxXXXXXXXXXXXXXXXX`（填入 `project.config.json` 的 `appid` 字段）
- **限制**：个人主体：1 个邮箱只能注册 1 个小程序，1 个身份证最多 5 个个人主体小程序

⚠️ **名称一年内只能修改 2 次**，想好再填。可先取个好记的名字，后续业务扩展了再改名。

### A3. 小程序类目（选错直接审核不通过）

类目必须在注册时选定，**部分类目需要额外资质（如《增值电信业务经营许可证》）**。以下是常用类目及资质要求：

| 你的小程序类型 | 推荐主类目 | 需要资质？ | 备选辅类目 |
|---------------|------------|------------|------------|
| **简历 / 证件照** | `工具 > 图片处理` | ❌ 个人可用 | `教育 > 职业培训`（需营业执照） |
| **健身运动** | `运动 > 健身` | ❌ 个人可用 | `工具 > 健康管理` |
| **生活服务（家政/维修/跑腿）** | `生活服务 > 家政` | ⚠️ 需营业执照 + 行业许可 | `生活服务 > 便民服务`（部分子类目个人可用） |
| **生活服务（水电缴费/社区）** | `生活服务 > 综合生活服务平台` | ⚠️ 需营业执照 | `工具 > 信息查询` |

> **关键规则**：类目选了「需要资质」的就必须上传资质文件，否则**审核直接拒绝**。建议初期全部选不需要资质的类目（如 `工具 > 图片处理`、`工具 > 信息查询`），等有营业执照后再补充。

### A4. 小程序备案（2023 年 9 月起强制）

不备案 = 不能提交审核上架。

| 步骤 | 操作 |
|------|------|
| 1 | 左侧「设置」→「基本设置」→ 找到「小程序备案」→ 点击「去备案」 |
| 2 | 填写主体信息（身份证/营业执照 + 住址 + 联系方式） |
| 3 | 填写小程序信息（名称与 A2 一致、服务内容描述 4-120 字、前置审批项如实填写） |
| 4 | 上传负责人身份证正反面 + 手持证件照（或法人授权书） |
| 5 | 提交审核 → 平台初审（**1-2 个工作日**）→ 工信部短信核验 → 管局终审（**3-20 个工作日**） |

- **耗时**：总计 **1-3 周**，建议越早做越好
- **关键**：工信部会发短信到负责人手机，**48 小时内必须点击链接核验**，否则流程中断重新提交
- **服务内容描述示例**：`本小程序为用户提供证件照在线编辑、AI 简历优化与模板套用功能，所有图片处理均在用户设备本地完成，不存储用户隐私数据。`

### A6. 隐私协议（审核必查项）

审核会检查「小程序是否在处理用户数据时有隐私政策」。需要在**云开发控制台**配置并发布。

| 步骤 | 操作 |
|------|------|
| 1 | 微信开发者工具 → 工具栏「云开发」→ 进入云开发控制台 |
| 2 | 左侧「设置」→「隐私与安全」→「隐私政策」→ 填写 |
| 3 | 内容模板（可直接用，把 `{你的小程序名称}` 替换掉）： |

```
{你的小程序名称} 隐私政策

1. 我们收集的信息
   本小程序在您使用时可能收集以下信息：
   - 您在编辑过程中上传的图片（仅用于证件照处理，处理完成后即时删除）
   - 微信授权登录信息（openid，用于识别用户身份和记录使用次数）
   - 使用行为数据（访问页面、停留时间，用于改进产品体验）

2. 信息的使用
   - 图片数据仅用于您发起的编辑操作（抠图/换底/美颜/导出），
     所有处理均在云端完成后立即删除，不会被用于其他目的。
   - openid 用于记录每日免费使用次数。
   - 不会将您的任何数据分享给第三方。

3. 信息的存储
   - 图片文件：处理完成后即时从云存储中删除。
   - 用户数据：存储在微信云开发数据库，仅您本人可访问。
   - 数据保留时长：您删除小程序后，数据同步清除。

4. 您的权利
   - 您可以在「我的」页面查看和删除您的历史作品。
   - 您可以随时通过微信删除小程序来清除所有关联数据。

5. 联系我们
   - 如有疑问，请联系开发者：[你的邮箱]
```

| 4 | 填写完毕后，点击「发布」→ 在小程序页面中**必须有入口能点开这个隐私协议**（通常放「我的」→ 关于） |

### B3. 腾讯云混元大模型 API 密钥

**需要 3 个值**：`SecretId`、`SecretKey`、`AppId`

#### 3.1 获取 AppId（纯数字）

🔗 [https://console.cloud.tencent.com/](https://console.cloud.tencent.com/) → 右上角头像 →「账号信息」→ 拉到「基本信息」→ 复制 **APPID**（如 `1234567890`）

#### 3.2 获取 SecretId / SecretKey

🔗 [https://console.cloud.tencent.com/cam/capi](https://console.cloud.tencent.com/cam/capi) →「新建密钥」→ 弹窗验证身份 → 生成后**立即下载 CSV 备份**（SecretKey 只显示一次）。

> ⚠️ 如果已有密钥但忘了 SecretKey，只能**删除后重建**。建议新建一个专用于混元的子账号密钥（CAM → 用户 → 新建 → 编程访问 → 只勾选混元相关权限），避免泄露主账号密钥。

#### 3.3 开通混元服务

🔗 [https://console.cloud.tencent.com/hunyuan](https://console.cloud.tencent.com/hunyuan) → 点击「开通服务」→ 后付费（按量计费）。**开通本身免费，实际调用才计费**，且前 50 万 token 免费。

#### 3.4 填写

**文件** `cloudfunctions/_shared/config.local.js`：
```javascript
HUNYUAN: {
  SECRET_ID: 'AKIDxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',
  SECRET_KEY: 'xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',
  APP_ID: '1234567890',
  ENDPOINT: 'hunyuan.tencentcloudapi.com',
  MODEL_PARSE: 'hunyuan-standard',
  MODEL_OPTIMIZE: 'hunyuan-pro',
},
```

## Part C — 部署 → 审核 → 上线（实操流水线）


### Step 5：提交审核

🔗 [mp.weixin.qq.com](https://mp.weixin.qq.com/) → 左侧「版本管理」→ 找到刚上传的版本 →「提交审核」

**审核前自检（100% 被拒项，必须查）**：

| 检查项 | 说明 |
|--------|------|
| ☐ 类目与功能匹配 | 上图处理类目 → 功能里不能出现社交/电商功能 |
| ☐ 隐私协议已发布且可访问 |「我的」→「关于」→ 可打开隐私政策页 |
| ☐ 没有测试数据/占位文本 | 页面里不能出现 `TODO`、`测试`、`Placeholder` |
| ☐ 所有页面都可正常操作 | 没有空白页、死循环、点不动、闪退 |
| ☐ 没有提交后跳转外部链接 | 不能在小程序内跳转其他小程序或 H5 支付页（除非是己方关联） |
| ☐ 用户授权提示清晰 | 需要相册/相机权限时有明确的用途说明 |
| ☐ 功能完整可用 | 审核员会实操测试，AI 生成/抠图/导出必须能完成 |

**常见被拒原因**：
- 类目不匹配（占驳回 40%）
- 未配置隐私协议（占驳回 25%）
- 页面空白/功能不完整（占驳回 20%）
- 有测试数据/占位内容（占驳回 10%）
- 需要额外资质但未提供（如选了企业资质类目但没上传营业执照）

审核一般 **1-3 个工作日**。

### Step 6：审核通过 → 发布

审核通过后，在「版本管理」中找到通过审核的版本 → 点击「发布」。发布后**全量用户即可在微信中搜索到你的小程序**（前提：已完成 A5 微信认证，否则只能通过扫码/分享访问）。

## Part D — 多小程序复用指南（简历 / 健身 / 生活服务）

你的场景：要做多个小程序，共用同一套技术栈（CloudBase + 混元 + 微信云开发），但业务逻辑各不同。

### D1. 架构复用方案

```
                    ┌─ tencent-cloud-account ──────┐
                    │  混元 API (共用一个密钥对)     │
                    │  腾讯云 COS / CDN（如需）       │
                    └───────────────────────────────┘
          ┌──────────────┬──────────────┬──────────────┐
          │              │              │              │
     ┌────▼─────┐  ┌────▼─────┐  ┌────▼─────┐  ┌────▼─────┐
     │ CloudBase │  │ CloudBase │  │ CloudBase │  │ CloudBase │
     │ Env-A     │  │ Env-B     │  │ Env-C     │  │ Env-D     │
     │ (求职)     │  │ (健身)     │  │ (家政)     │  │ (未来...)  │
     └───────────┘  └───────────┘  └───────────┘  └───────────┘
          │              │              │              │
     ┌────▼─────┐  ┌────▼─────┐  ┌────▼─────┐  ┌────▼─────┐
     │ 小程序1   │  │ 小程序2   │  │ 小程序3   │  │ 小程序N   │
     │ AppID-A   │  │ AppID-B   │  │ AppID-C   │  │ AppID-N   │
     └───────────┘  └───────────┘  └───────────┘  └───────────┘
```

**核心原则**：
- 一个微信开放平台账号 → 多个小程序（个人主体：1 个身份证最多 5 个）
- 每个小程序 → 一个独立 CloudBase 环境（数据隔离、互不影响）
- 所有小程序 → 共用同一套腾讯云 API 密钥（混元、人像分割等）
- GitHub 组织/账号下分仓库管理，`_shared` 公共模块可抽成独立 npm 包

### D2. 新小程序启动清单（模板）

当你新建一个 xx 小程序时，按这个清单走：

| # | 事项 | 详细 |

### D4. 腾讯云账号复用

**一次开通，所有小程序通用**：
- 同一个腾讯云账号下的 API 密钥（SecretId/SecretKey）可以用在所有 CloudBase 环境的云函数中
- 混元、人像分割、语音识别等腾讯云服务也共用同一账号
- 各小程序的 CloudBase 环境是独立的（数据库/存储/函数各自管理），不用担心数据串

## Part E — 快速自检清单

### 每次部署前

```
☐ project.config.json 的 appid 不是占位符
☐ 根目录 config.js 的 CLOUDBASE_ENV_ID 不是占位符
☐ cloudfunctions/_shared/config.local.js 存在且 P0 三字段已填
☐ 微信开发者工具能正常编译、预览
☐ bash deploy.sh 跑通（15 云函数 + 5 集合）
☐ 本地测试：证件照抠图→换底→导出 全流程可走通
☐ 本地测试：简历导入→套模板→导出 PDF 全流程可走通
```

```

☐ 类目与功能匹配（无资质要求的类目优先）
☐ 无测试数据 / 占位文本 / TODO
☐ 所有页面无空白、无闪退
☐ 用户授权提示清晰（相机/相册用途说明）
☐ 小程序名称/头像/简介已填且合规
☐ 已完成备案（非企业主体个人备案可能较慢，提前做）
```
